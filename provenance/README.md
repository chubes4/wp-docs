# Provenance

This directory holds lightweight project-level provenance notes and schemas.

Large execution artifacts, screenshots, logs, and previews should stay with the generation runtime or artifact bundle that produced them. Reviewed content should keep enough page-level metadata to answer these questions:

- Which source repo and commit was used?
- Which source files were read?
- Which agent bundle and target set generated the draft?
- What review state is the page in?
- What changed since the previous accepted version?

The first shared source map target is `content/_meta/source-map.json`.
